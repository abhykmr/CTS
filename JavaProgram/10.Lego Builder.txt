Description

Objective:

To work with inheritance.

Concept Explanation:


Inheritance in object-oriented programming allows subclasses to inherit properties and behaviors from a superclass, promoting code reuse and extending functionality. It establishes class hierarchies, enabling subclasses to inherit and extend features from their super classes. This fosters flexible and modular code structures, facilitating the sharing and extension of code functionality across related classes.

Concept Implementation:

The code defines two subclasses, ElectricalItemDetails and PlumbingItemDetails, inheriting from the super class LegoBuilders. Each subclass contains specific attributes and a method to calculate the total bill based on product type. The UserInterface class prompts the user for product details, creates the appropriate subclass object, and calculates the total bill. This implementation showcases inheritance and encapsulation in Java. 

Scenario:

A new construction company named Lego Builders. They aim to automate the generation of bills based on the Product Type. They have asked you to develop software to incorporate the feature to generate the bill amount as their software consultant.

Requirements 1:  Calculation of bill amount

The application is to calculate the bill amount based on the product type.
